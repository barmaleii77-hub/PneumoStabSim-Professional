# Beta testing playbook

## Циклы обратной связи

1. **Сбор отчётов** — тестеры заполняют форму `Обратная связь` в приложении
   (QML-панель `FeedbackForm.qml`). Данные попадают в `reports/feedback/inbox.jsonl`
   через сервис `src/services/feedback_service.py`.
2. **Аналитика** — сервис обновляет агрегированные метрики (`summary.json`,
   `summary.md`). Скрипт `tools/feedback_sync.py summary` можно запускать вручную
   для восстановления итогов перед митингами.
3. **Синхронизация** — команда запускает `tools/feedback_sync.py push` (с
   необходимыми переменными окружения) для отправки накопленных заявок в
   issue-трекер. После успешной выгрузки записи помечаются в `sent.log`.
4. **Решение** — инженеры анализируют тикеты, комментируют статус и при
   необходимости запрашивают дополнительную информацию у автора (контакты
   передаются из формы).

## Критерии завершения тестирования

- Все критические и блокирующие отчёты из `summary.md` переведены в состояние
  «решено» в трекере.
- Для отчётов средней важности сформированы подтверждённые планы исправления
  (issue с назначенным исполнителем и дедлайном).
- Минимум один полный цикл синхронизации (push + summary) проведён за последние
  48 часов.
- Документация по найденным регрессиям обновлена в соответствующих руководствах
  (UI, графика, симуляция).
