# check_git_status.ps1 — руководство по использованию

## Назначение
`check_git_status.ps1` — скрипт PowerShell для интерактивной проверки состояния Git. Версия 2.0.0 поддерживает локальные клоны без настроенных remotes и предоставляет рекомендации по синхронизации.

> ℹ️ Для быстрых проверок синхронизации без подробного интерактива используйте обновлённый скрипт [`check_git_sync.ps1`](./check_git_sync.md). Он фиксирует платформу, безопасно обрабатывает отсутствие `origin` и сообщает, как создать удалённую ветку.

## Быстрый старт
```powershell
pwsh ./check_git_status.ps1
```

### Вывод включает
- текущую ветку и последний коммит;
- рабочий статус (`git status --short`), изменённые и неотслеживаемые файлы;
- список remotes и информацию об upstream (если настроен);
- подсказки по `git pull`/`git push` и фиксации изменений.

## Поведение в разных сценариях
| Сценарий | Поведение скрипта |
|----------|-------------------|
| Нет remotes | Показывает предупреждение и предлагает настроить remote |
| Есть remote, но нет upstream | Сообщает об отсутствии отслеживаемой ветки и пропускает сравнение |
| Настроен upstream | Отображает количество коммитов впереди/позади |
| Изменения в рабочем каталоге | Перечисляет изменённые и неотслеживаемые файлы |

## Рекомендованный рабочий процесс
1. Запуск скрипта перед коммитами и перед релизом.  
2. Настройка remote: `git remote add origin <url>`.  
3. Перезапуск скрипта для получения подсказок по push/pull.  
4. После push сохранить отчёт (обновить документы в `archive/2025-11/root-reports/`).

## Требования
- PowerShell 7+ (или Windows PowerShell 5.1).  
- Git в `PATH`.
