# UAT чек-листы: панель взаимодействий и валидации UI

## Цели
- Подтвердить, что модульные панели графики синхронизируются с настройками без побочных эффектов.
- Проверить, что новые валидации UI корректно блокируют неверные состояния и сообщают об ошибках.
- Зафиксировать ручные сценарии UAT для регрессионного повторения.

## Чек-лист
- [x] Инициализация GraphicsPanel с временным конфигом не модифицирует основной `config/app_settings.json`.
- [x] Неверные категории (`unexpected`) отклоняются через `GraphicsSettingsService.ensure_valid_state`.
- [x] Нарушение диапазонов окружения (fog_far < fog_near) приводит к ошибке `EnvironmentValidationError`.
- [x] Валидированный снапшот сохраняется и читается повторно через `save_current`/`load_current` без потери данных.
- [ ] Проверить визуальные подсказки для кнопки валидации (ручной сценарий).
- [ ] Подтвердить отображение сообщений об ошибках в UI при неправильном вводе (ручной сценарий).

## Ручные сценарии
1. **Валидация окружения через UI**  
   - Запустить приложение в тестовом профиле.  
   - Открыть вкладку «Окружение», ввести `fog_far` < `fog_near`.  
   - Нажать «Проверить» — ожидание: блокирующее уведомление об ошибке, поля подсвечены.  
   - Результат: _не выполнено (ожидает ручного прогона)._ 

2. **Сохранение валидированного состояния**  
   - Внести изменения в скорость камеры и MSAA, нажать «Сохранить».  
   - Перезапустить приложение, убедиться, что значения загружены.  
   - Результат: _не выполнено (ожидает ручного прогона)._ 

3. **Отказ при неизвестной категории**  
   - Через консоль разработчика отправить патч с ключом `unexpected`.  
   - Ожидание: событие журнала с категорией ошибки, состояние не сохраняется.  
   - Результат: _не выполнено (ожидает ручного прогона)._
