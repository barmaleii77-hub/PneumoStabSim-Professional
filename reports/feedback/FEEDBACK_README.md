# Система обратной связи - Микрошаги UI

## ?? Где смотреть логи и отчёты

### Структура артефактов:
```
reports/
??? ui/              # Отчёты по UI изменениям
?   ??? audit_pre.md      # Пред-аудит состояния
?   ??? audit_post.md     # Пост-аудит (будет создан)
??? feedback/        # Управление процессом
?   ??? CONTROL_PLAN.json # План выполнения микрошагов
?   ??? FEEDBACK_README.md # Эта инструкция
??? tests/           # Результаты тестов

artifacts/
??? ui/              # Артефакты UI
?   ??? widget_tree_pre.json  # Дерево виджетов до изменений
?   ??? widget_tree_post.json # Дерево виджетов после изменений
??? diffs/           # Патчи изменений

logs/
??? ui/              # Логи изменений UI
?   ??? ui_params.log      # Изменения параметров
?   ??? push.log          # Логи сохранения в VCS
??? tests/           # Логи тестов
```

## ?? Как давать обратную связь

### 1. Проверить прогресс:
Смотрите `reports/feedback/CONTROL_PLAN.json` для статуса выполнения микрошагов.

### 2. Просмотреть изменения:
- **До изменений:** `reports/ui/audit_pre.md`
- **После изменений:** `reports/ui/audit_post.md` (будет создан)
- **Конкретные диффы:** `artifacts/ui/diff_*.patch`

### 3. Проверить тесты:
- **Результаты тестов:** `reports/tests/`
- **Логи тестов:** `logs/tests/`

### 4. Отследить параметры:
- **Изменения UI:** `logs/ui/ui_params.log`
- **Текущее состояние:** `artifacts/ui/ui_state.json`

## ?? Контакты для обратной связи

При обнаружении проблем проверьте:
1. `reports/feedback/CONTROL_PLAN.json` - текущий статус
2. `reports/ui/audit_*.md` - анализ изменений  
3. `logs/ui/` - детальные логи
4. `artifacts/ui/` - снимки состояния
